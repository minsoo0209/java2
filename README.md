## for-each문

### 기본 문법
```
for (타입 변수명 : 배열이나_컬렉션) {
    //반복해서 실행할 코드
}
```
### 주요 특징 및 장점
- 가독성 : 코드가 짧아져서 읽기 쉽다.
- 안전성 : 인덱스(i)를 직접 다루지 않아 배열의 범위를 벗어나는 실수를 방지 가능
- 제한사항 : 요소를 읽기 전용으로 가져오기 때문에, 반복문 안에서 배열의 값을 직접 수정하거나 인덱스 번호를 알아내야 할 때는 일반 for문을 쓰는게 좋다.


## 2차원 배열

- 배열의 선언 : 배열을 만들겠다고 변수를 선언하는 단계 []를 두개 씀
```
int intAraay[][];
or
int[][] intAraay;     //2차원 배열의 레퍼런스 변수 선언 하는법 두가지
```
- 배열 생성 : 실제 메모리에 공간을 할당하는 단계
```
intAraay = new int[2][5]; //2행(세로줄) 5열(가로칸) 2x5의 2차원 배열 생성
```
- [그림3-14]에서 2차원 배열의 length 의미
- i.length -> 2차원 배열의 행의 개수, 2
- i[0].length -> 0번째 행의 열의 개수, 5
- i[1].length -> 1번째 행의 열의 개수, 5

### 2차원 배열의 초기화
- 다음과 같이 2차원 배열을 선언할 때 각 원소를 초기화할 수 있다. 이 때 자동으로 초기화된 배열이 생성된다.
```
int intArray[][] = { {0,1,2}, {3,4,5}, {6,7,8} }; //3x3 배열 생성
char charArray[][] = { {'a','b','c'}, {'d','e','f'} }; //2x3 배열 생성
double doubleArray[][] = { {0.01,0.02}, {0.03,0.04} }; //2x2 배열 생성
```


## 메소드의 배열 리턴

### 핵심개념
- 배열의 전체가 복사되어 전달되는 것이 아니라, 배열이 어디 있는지 알려주는 주소값(레퍼런스)만 리턴된다.

### 메소드의 리턴 타입 선언 규칙
- 타입일치 : 리턴하는 배열의 타입과 리턴을 받는 변수의 타입이 반드시 같아야한다.
- 크기 지정 금지 : 리턴 타입 선언 시 int[4]처럼 크기를 써넣으면 안된다. 그냥 int[]라고만 써야한다.

### 배열을 리턴 받아 사용하는 과정
```
int[] makeArray() {

int temp[] = new int[4];

return temp;

}
```
- int[] (리턴타입) : 이 메소드는 실행이 끝나면 정수들이 담긴 배열을 결과물로 준다는 의미
- makeArray() (메소드이름) : 이 기능의 이름
- int temp[] = new int[4] : 메소드 안에서 4칸짜리 새로운 배열을 만든다. temp는 그 배열을 가리키는 임시 이름표
- return temp; : 만든 배열의 주소값(레퍼런스)을 메소드를 호출한 쪽으로 던져준다.



```
int[] intArray
```
- (1) 레퍼런스 변수 선언
- 배열을 가르킬 이름표(intArray)를 먼저 만든다.
- 아직은 아무것도 가리키지 않은 상태

```
makeArray();
```
- (2) 메소드 실행 및 배열 생성
- 메소드가 실행되면서 메모리에 실제

```
intArray - makeArray();
```
- (3) 리턴값(주소) 치환
- 메소드가 종료되면서 temp가 가지고 있떤 배열의 주소값을 intArray에게 넘겨준다
- intArray는 메소드 밖에서도 해당 배열을 똑같이 가리키게 된다.

```
for (int i=0; i < intArray.length; i++) { intArray[i] = i; }
```
- (4) 배열 요소 사용
- 이제 intArray를 이용해 실제 배열 공간에 값을 넣거나 읽을 수 있다

## 자바의 예외 처리

### 핵심 개념
- 예외 : 실행 중 오동작이나 결과에 악영향을 미치는 예상치 못한 상황 발생
- 자바의 예외 처리 : 프로그램 실행 중 예기치 않게 발생하는 에러에 대비하여, 프로그램이 갑자기 멈추지않고 정상적으로 작동하거나 안전하게 종료될 수 있도록 코드를 작성하는것을 말한다.

### try-cat-finally문
```
trt {
    예외가 발생할 가능성이 있는 실행문(try 블록)
}
catch (처리할 예외 타입 선언) {
    예외 처리문(catch 블록)
}
finally {
    예외 발생 여부와 상관없이 무조건 실행되는 문장(finally 블록)
}
```
### 1.구조와 역할
- try
- 역할 : 예외가 발생할 가능성이 있는 코드를 작성한다.
-  실행조건 : 항상 가장 먼저 실행된다.

- catch
- 역할 : 발생한 예외를 처리하는 코드를 작성한다.
- 실행조건 : try블록에서 예외가 발생했을 때만 실행된다.

- finally
- 역할 : 예외 발생 여부와 상관없이 무조건 실행되어야 하는 코드를 작성한다.
- 실행조건 : 항상 마지막에 실행된다.

### 2.동작 흐름
- 예외가 발생하지 않을 때 : try 실행 -> finally 실행 (정상종료)
- 예외가 발생했을 때 : try 실행 중 중단 -> catch 실행(예외처리) -> finally 실행

### 자바의 예외 클래스
- 자바는 응용프로그램이 실행 중 오류를 탐지할 수 있또록 많은 예외를 클래스 형태로 제공

| 예외 타입(클래스) | 예외 발생 경우 | 패키지 |
| :--- | :--- | :--- |
| **ArithmeticException** | 정수를 0으로 나눌 때 발생 | `java.lang` |
| **NullPointerException** | null 레퍼런스를 참조할 때 발생 | `java.lang` |
| **ClassCastException** | 변환할 수 없는 타입으로 객체를 변환할 때 발생 | `java.lang` |
| **OutOfMemoryError** | 메모리가 부족한 경우 발생 | `java.lang` |
| **ArrayIndexOutOfBoundsException** | 배열의 범위를 벗어난 접근 시 발생 | `java.lang` |
| **IllegalArgumentException** | 잘못된 인자 전달 시 발생 | `java.lang` |
| **IOException** | 입출력 동작 실패 또는 인터럽트 시 발생 | `java.io` |
| **NumberFormatException** | 문자열이 숫자 형식이 아닐 때 변환 시 발생 | `java.lang` |
| **InputMismatchException** | Scanner 사용 시 타입이 맞지 않는 값을 입력할 때 발생 | `java.util` |

### 예외 클래스 사례
```
int[] intArray = new int[5]; // 크기가 5인 배열을만듬


try {
    intArray[3] = 10; // 예외 발생하지않음
    intArray[6] = 5; // 예외 발생(인덱스 범위는 0~4)
}
catch (ArrayIndexOutOfBoundsException e) { // 객체 e에 예외 정보가 넘어옴
    System.out.println("배열 범위 초과"); //
}
```

## 자바의 객체 지향 특성

### (1)캡슐화
- 캡슐화 : 객체를 캡슐로 싸서 내부를 볼 수 없게 하는 것
- 객체의 가장 본질적인 특징
- 외부의 접근으로부터 객체 보호

### (1-1)자바의 캡슐화
- 클래스 : 객체 모양을 선언하는 틀(캡슐화하는 틀)
- 객체 : 생성된 실체(instance) : 클래스 내에 메소드와 필드 구현

### (2)상속
- 상위 객체의 속성이 하위 객체에 물려 줌
- 하위 객체가 상위 객체의 속성을 모두 가지는 관계
- [예시] 사람은 생물의 속성은 가지지만 식물의 속성은 가지지 않음.

### (2-1)자바의 상속
- 상위 클래스의 멤버를 하위 클래스가 물려받음
- 상위클래스 : 수퍼 클래스
- 하위 클래스 : 서브 클래스, 수퍼 클래스 코드의 재사용, 새로운 특성 추가 가능

### (3)다형성
- 다형성 : 같은 이름의 메소드가 클래스 혹은 객체에 따라 다르게 구현 되는 것
- [예시]
- 메소드 오버로딩 : 한 클래스 내에서 같은 이름이지만 다르게 작동하는 여러 메소드
- 메소드 오버라이딩 : 슈퍼 클래스의 메소드를 동일한 이름으로 서브 클래스마다 다르게 구현

### 객체 지향 언어의 목적
- (1)소프트 웨어의 생산성 향상
- 컴퓨터 산업 발전에 따라 소프트웨어 생명 주기 단축
- 소프트웨어를 빠른 속도로 생산할 필요성 증대
- (2)객체 지향 언어
- 상속, 다형성, 객체, 캡슐화 등 소프트웨어 재사용을 위한 여러 장치 내장
- 소프트웨어 재사용과 부분 수정 빠름
- 소프트웨어를 다시 만드는 부담 대폭 줄임
- 소프트웨어 생산성 향상
- (3)실세계에 대한 쉬운 모델링
- 초기 프로그래밍
- 수학 계산/통계 처리를 하는 등 처리 과정, 계산 절차 중요
- (4) 현대 프로그래밍
- 컴퓨터가 산업 전반에 활용
- 실세계에서 발생하는 일을 프로그래밍
- 실세계에서는 절차나 과정보다 물체들의 상호 작용으로 묘사하는 것이 용이
- (5) 객체 지향 언어
- 실세계의 일을 보다 쉽게 프로그래밍하기 위한 객체 중심적 언어

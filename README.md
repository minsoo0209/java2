## 8장. 자바 GUI 스윙(Swing)

### 1. GUI와 스윙(Swing)의 개념
- AWT (Abstract Window Toolkit): 운영체제(OS) 고유의 UI 패키지를 차용하여 렌더링하므로 OS마다 모양이 다르게 출력되는 중량(Heavyweight) 컴포넌트입니다.
- Swing: 자바 소스코드로 직접 픽셀을 그려서 출력하는 독립적 UI 패키지입니다. 어떤 OS에서 실행되든 일관되고 예쁜 디자인을 보장하는 경량(Lightweight) 컴포넌트이며, 모든 클래스명이 J로 시작합니다 (JFrame, JButton, JLabel 등).

### 2. 컴포넌트와 컨테이너 개념
자바 GUI 화면을 구성하는 핵심 수직 관계 계층입니다.
- 상속 구조: Component (AWT) -> Container (AWT) -> JComponent (Swing)
- 컴포넌트 (Component): 버튼, 텍스트 상자, 라벨 등 화면에 직접 표시되고 사용자와 직접 소통하는 독립적인 UI 부품입니다. 홀로 화면에 노출될 수 없으며 반드시 컨테이너에 포함되어야 합니다.
- 컨테이너 (Container): 다른 컴포넌트들을 자신의 영역 안에 감싸고 배치할 수 있는 그릇 역할의 컴포넌트입니다. 컨테이너 역시 컴포넌트의 일종이므로 컨테이너 안에 또 다른 컨테이너를 부착하는 중첩 배치가 가능합니다.

### 3. 최상위 컨테이너 (Top-level Container)
다른 컨테이너에 속하지 않고, 독립적으로 화면에 출력될 수 있는 가장 바깥쪽의 메인 창(윈도우)을 의미합니다. 스윙 프로그램 작동을 위해서는 필수적입니다.
- JFrame: 가장 보편적으로 사용되는 윈도우 창으로, 타이틀바, 최소화/최대화/닫기 버튼을 기본 제공합니다.
- JDialog: 주로 알림 메시지, 경고, 패스워드 입력 등을 위한 팝업 대화상자 창을 생성할 때 사용합니다.
- JApplet: 웹 브라우저 내부에서 임베디드되어 동작하는 스윙 창입니다 (현재는 브라우저 보안 이슈로 유실되어 거의 쓰이지 않음).

### 4. 프레임에 컴포넌트 부착하기 (콘텐츠팬 메커니즘)
최상위 컨테이너인 JFrame에 컴포넌트를 올릴 때 작동하는 내부 화면 구조 레이어입니다.

1. 콘텐츠팬(ContentPane) 설명: JFrame 객체가 생성될 때 내부에 자동으로 장착되는 기본 영역 컨테이너입니다. 프레임 윈도우에서 상단 타이틀바와 테두리를 제외하고 실제 UI 요소들이 뿌려지는 핵심 영역입니다.
2. 콘텐츠팬 알아내기: 프레임 내부에서 컴포넌트를 붙이거나 배치 규칙을 지정하려면 먼저 getContentPane() 메소드를 호출하여 콘텐츠팬의 주소를 참조해야 합니다.
3. 콘텐츠팬에 컴포넌트 붙이기: 알아낸 콘텐츠팬의 주소 레퍼런스(cp)를 통해 cp.add(컴포넌트) 형태로 부품을 부착합니다.
4. 콘텐츠팬 변경(통째로 바꾸기): 화면 전체를 전환하거나 다른 레이아웃으로 덮어쓸 때, 컴포넌트를 하나씩 지우지 않고 새롭게 제작한 JPanel 등을 setContentPane(새로운_패널) 메소드를 통해 통째로 교체할 수 있습니다.

### 5. 배치 관리자 (Layout Manager) 유형 4가지
컨테이너 영역 내부에 부착되는 컴포넌트들의 위치와 크기를 자동으로 조절하고 정렬해 주는 내장 관리 객체입니다.

| 배치 관리자명 | 컴포넌트 정렬 방식 및 특징 | 기본 탑재 컨테이너 |
| :--- | :--- | :--- |
| FlowLayout | 컴포넌트를 삽입 순서대로 좌 -> 우 정렬합니다. 공간이 부족하면 자동으로 개행하여 상 -> 하로 배치합니다. 컴포넌트는 자신이 가지고 있는 본래의 고유 크기(Preferred Size)를 잃지 않고 유지합니다. | JPanel |
| BorderLayout | 컨테이너의 면적을 동(EAST), 서(WEST), 남(SOUTH), 북(NORTH), 중앙(CENTER) 5개 영역으로 분할 배치합니다. 컴포넌트를 추가할 때 구역을 반드시 명시해야 하며, 지정하지 않을 시 기본값인 CENTER에 집중됩니다. 중앙 구역은 남은 잔여 공간을 채우기 위해 사방으로 무조건 확장됩니다. | JFrame 콘텐츠팬 |
| GridLayout | 화면 영역을 가로, 세로 정해진 비율의 행과 열로 쪼개어 바둑판 모양의 격자를 구성합니다. 컴포넌트가 들어가는 순서대로 위에서 아래로 칸을 채우며, 배치된 모든 컴포넌트의 크기가 완벽하게 동일하게 균등 배분됩니다. (예: 계산기 숫자 버튼) | 없음 (개발자가 명시적 지정) |
| CardLayout | 컴포넌트들을 마치 포커 카드 덱처럼 차곡차곡 위로 포개어 배치하는 정렬 방식입니다. 한 번에 여러 레이어가 출력되지 않고 오직 하나의 카드(화면)만 표면에 노출됩니다. 이벤트 리스너와 연결하여 다음 화면으로 슬라이드 전환을 시도할 때 적극 차용됩니다. | 없음 (개발자가 명시적 지정) |

💡 배치 관리자가 없는 컨테이너 구현 (null 지정)
개발자가 격자나 방향에 얽매이지 않고 임의의 절대 좌표(x, y)와 크기(width, height)를 수치로 직접 지정하여 컴포넌트를 고정 배치하고 싶다면 cp.setLayout(null); 명령어로 배치 관리자를 해제해야 합니다. 배치 관리자를 삭제한 뒤에는 반드시 각 컴포넌트마다 setBounds(x, y, w, h); 메소드를 명시해 주어야 화면에 출력됩니다.

## 6장. 모듈, 패키지 및 기본 클래스 활용

### 1. 패키지(Package)와 import
- 클래스를 기능별로 묶어 관리하는 디렉터리 구조로, 클래스 이름 충돌을 방지합니다.
- 외부 패키지의 클래스를 가져올 때는 `import` 문을 사용하여 경로를 명시합니다.

### 2. Wrapper 클래스와 박싱/언박싱
- **Wrapper 클래스**: int, char 등 기본 타입을 객체로 다루기 위해 제공되는 클래스입니다.
- **박싱(Boxing)**: 기본 타입의 값을 대응하는 Wrapper 객체로 변환하는 과정입니다.
- **언박싱(Unboxing)**: Wrapper 객체에서 값을 꺼내 기본 타입으로 변환하는 과정입니다.
- **자동 변환**: 컴파일러가 상황에 맞춰 자동으로 박싱과 언박싱을 처리해 줍니다.

### 3. Math 클래스
- 모든 메소드가 정적(static)으로 선언되어 객체 생성 없이 클래스명으로 호출합니다.
- `Math.random()`: 0.0 이상 1.0 미만의 무작위 난수를 생성합니다.
- `Math.abs()`: 수치의 절대값을 계산합니다.
- `Math.sqrt()`: 수치의 제곱근을 계산합니다.

### 4. Object 클래스
- 모든 자바 클래스의 최상위 조상 클래스입니다.
- `equals()`: 객체의 내용이 같은지 비교하기 위해 자주 재정의하여 사용합니다.
- `toString()`: 객체를 문자열로 표현할 때 사용하며, 보통 클래스 정보를 출력하도록 오버라이딩합니다.

## 7장. 제네릭과 컬렉션

### 1. 컬렉션 프레임워크(Collection Framework)
- 가변 크기의 컨테이너로, 배열의 크기 고정 문제를 해결한 구조입니다.
- **ArrayList<E>**: 인덱스를 사용해 객체를 순차적으로 관리하는 동적 배열입니다.
- **HashMap<K, V>**: 키(Key)와 값(Value)을 쌍으로 묶어 관리하며, 키를 이용한 검색 성능이 뛰어납니다.

### 2. 제네릭(Generic)
- 클래스나 메소드에서 사용할 데이터 타입을 파라미터화하여 결정하는 기법입니다.
- 컴파일 시점에 엄격하게 타입을 체크하여 타입 안정성을 보장하고 불필요한 형 변환을 줄여줍니다.



## 04.29 9주차

## 주제: 상속 (Inheritance)

### 1. 상속의 기초 (Inheritance Basics)
상속은 기존 클래스를 재사용하여 새로운 클래스를 작성하는 자바의 핵심 기술입니다.

- 키워드: extends를 사용하여 부모 클래스를 지정합니다.
- 특징:
    - 자바는 단일 상속만 지원합니다.
    - 상속을 받으면 부모의 필드와 메소드를 물려받아 사용할 수 있습니다.
    - super()를 통해 부모 클래스의 생성자를 호출합니다.

### 2. 메소드 오버라이딩 (Method Overriding)
부모 클래스로부터 물려받은 메소드를 자식 클래스에서 새롭게 재정의하는 기능입니다.

- 동적 바인딩: 부모 타입으로 선언된 레퍼런스라도 실제 객체가 자식 객체라면, 자식의 오버라이딩된 메소드가 실행됩니다.

### 3. 추상 클래스 vs 인터페이스
구분 | 추상 클래스 (Abstract Class) | 인터페이스 (Interface)
:--- | :--- | :---
키워드 | abstract class | interface
구성 | 추상 메소드 + 일반 메소드 | 주로 추상 메소드 + 상수
목적 | 기능 확장 및 상속 계층 형성 | 클래스 간의 동일한 규격 정의
구현 | extends (단일 상속) | implements (다중 구현 가능)

### 4. 핵심 코드 예제 (Point & ColorPoint)
```java
class Point {
    private int x, y;
    public Point(int x, int y) { this.x = x; this.y = y; }
    public void show() { System.out.println("(" + x + "," + y + ")"); }
}

class ColorPoint extends Point {
    private String color;
    public ColorPoint(int x, int y, String color) {
        super(x, y); // 부모 생성자 호출
        this.color = color;
    }
    @Override
    public void show() { // 오버라이딩
        System.out.println(color + "색 점 (" + x + "," + y + ")");
    }
}
